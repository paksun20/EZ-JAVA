[추상 클래스]

(추상)
1. 실체 간에 공통된는 특성을 추출한 것이다.
2. 구체적인 실체라기보다는 실체들의 공통되는 특성을 가지고 있는 추상적인 것

(추상클래스)
1. 클래스들의 공통적인 특성를 추출해서 선언한 클래스
2. 추상 클래스와 실체 클래스는 상속의 관계를 가진다.
3. 추상 클래스는 부모이고 실체는 자식으로 구현된다.
4. 추상 클래스는 객체를 생성해서 사용할 수 없다.
4. 반드시 상속을 통해서 자식 클래스에서 객체를 생성해야 한다.

(특성)
1. 클래스에 정의된 필드(field) : 데이터를 담는 그릇
2. 클래스에 정의된 메소드(method) : 그릇에 담긴 데이터를 어떻게 처리할 것인가?
3. 클래스는 특성들을 모아놓은 큰 그릇 

(실례)
1. 새(Bird), 곤충(Insect), 물고기(Fish)
2. 공통적인 특성은?
3. 보는 관점?
   - 어떻게 처리할 것인가?
   - 목적 : 요리재료, 관리대상, 생물학적인 분류
4. 새(Bird), 곤충(Insect), 물고기(Fish) -> 동물(Animal) 

(추상 클래스의 용도)
1. 추상 클래스를 만들어서 사용하는 이점(장점)?
   - 실체 클래스들의 공통된 필드와 메소드의 이름을 통일
   - 실체 클래스의 작성할 시간을 절약
   
2. 새(Bird)
   - 날다. 걷다
   - class Bird {
	    void fly() {}
     }
     
3. 곤충(Insect)
   - 날다. 기다
   - class Insect {
        void  fly() {}
     }
   
   
4. 물고기(Fish)
   - 헤엄친다. 
   - class Fish {
	    void swim() {}
     }   

5. 동물(Animal)
   - 움직인다.
   - class Animal {
        void move() {}
     }
