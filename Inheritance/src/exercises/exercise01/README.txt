[확인문제1]

1. 자바의 상속에 대한 설명 중 틀린 것은?
정답: 1번
해설:
1) 자바는 다중 상속을 혀용한다.
   - 부모는 하나만 있어야 한다.
   
2) 부모의 메소드를 자식 클래스에서 재정의(오버라이딩)할 수 있다.
   - @Override : 컴파일러 지시자(annotation)
   
3) 부모의 private 접근 제한을 갖는 필드와 메소드는 상속의 대상이 아니다.
   - public나 protected 만 접근할 수 있다.
   
4) final 클래스는 상속할 수 없고, final 메소드는 오버라이딩 할 수 없다.
   - final은 더 이상 상속이나 메소드 오러라이딩 제한을 할 때 사용

// 다중상속 허용하지 않음
// 부모는 하나민 있어야 한다.
class Child extends Parent {

}
