[중첩 클래스와 충첩 인터페이스]

(목적)
1. 외부에 공개할 필요가 필요가 없는 클래스를 내부 클래스로 만들어 사용함으로써 외부 인터페이스를 명료
2. 외부로 공개할 부분과 내부에서만 사용해야 하는 기능을 분리
3. 내부에서 사용하는 전용 클래스로 한정 

(중첩 클래스)
1. 클래스 안에 클래스를 정의
2. 멤버 클래스
   - 인스턴스 멤버 클래스(class)
   - 정적 멤버 클래스(static class)
2. 로컬 클래스(클래스의 메소드 안에 정의 된 클래스)   

(중첩 인터페이스)
1. 클래스 안에 인터페이스를 정의
2. 인스턴스 멤버 클래스
